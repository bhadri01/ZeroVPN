01 What is it

A control plane for your own WireGuard network.

ZeroVPN manages devices, peers, IP allocation, and admin policy for a WireGuard network you host yourself — and puts a live console over the whole fleet. A Rust API and worker, a React single-page app, and one Docker Compose stack. No accounts on someone else's servers.

It's a private-network access VPN — each device split-tunnels only the VPN's own subnet (10.10.0.0/22 by default), so it connects your people to your machines. It is deliberately not an anonymity or full-tunnel product.

P/01

Homelab tinkerer

One box in the basement, a handful of devices, a NAS, a Pi. Five-minute Docker bootstrap, QR onboarding, live topology on the TV.

P/02

Small team

Ten to fifty peers with real users and real auditors. Suspend, quota, and key-rotate from an admin console; audit log with CSV export.

P/03

Compliance operator

You need to answer "who did what, from where" on demand. Per-user sign-in and audit trail; per-device handshake and endpoint history.

02 Features

The boring parts. Handled.

Everything below ships in the current codebase — nothing here is a mockup or a promise.

02.1 For everyone

Onboarding

Device provisioning

Server-generated keypairs, IP allocation, and a ready .conf plus QR to scan into the WireGuard app.

Live

Real-time telemetry

Worker → ZeroMQ → API → WebSocket at ~1 Hz. Samples are stored and replayed on refresh, so the chart is never empty on reload.

Map

Topology & flows

A two-level tree that mirrors how WireGuard thinks. Toggle to a live flows view of active connections. Layout persists per user.

Per-device

Stats & lifecycle

Per-device bandwidth, handshake and endpoint history, and a timeline of every lifecycle event from creation to revoke.

2FA

TOTP & recovery codes

RFC 6238 authenticator support with hashed one-time recovery codes, opt-in per account.

Client

Installable PWA

Install to the home screen; in-app and OS notifications for connectivity, quota, security, and device lifecycle events.

02.2 For admins

Users

Full user control

Suspend, set quotas, rotate keys, impersonate for support, and page through every account from one board.

Fleet

Servers & sessions

Manage WireGuard hub servers, review the session & security event log (logins, 2FA, impersonation), and view a fleet-wide topology across every user.

Trail

Audit & access logs

Append-only audit log (filterable, with CSV export), a failed-login board, and filterable access logs.

Ops

Quotas & maintenance

Per-user bandwidth quotas with auto-pause, plus a maintenance mode that gates the app during upkeep.

03 How it works

One stack. Every hop is yours.

Five well-defined layers from browser to backbone. No third-party SaaS in the data path, and no telemetry phoning home.

Edge

Traefik

TLS termination and automatic HTTPS via ACME. The only port open to the internet besides WireGuard's UDP.

Control

API · Rust/axum

Auth, devices, admin, and mail. It also brings up the WireGuard interface in its own container — no separate wg sidecar — and serves the OpenAPI spec.

Worker

Poller · ZeroMQ

Polls WireGuard peer stats (~1 Hz), rolls up bandwidth, and publishes tick-level samples over a ZeroMQ socket.

Data

PostgreSQL 18

Sessions, devices, samples, and every application record. The single source of truth for the whole stack.

Resolver

CoreDNS

Serves per-peer DNS from a hosts file the API maintains, so devices resolve each other by friendly name.

Wire

WireGuard · userspace

The API brings up wg0 with userspace boringtun — no host kernel module, no modprobe.

Live-stats path: the worker samples peers and publishes over ZeroMQ; the API relays those frames to the browser over a WebSocket (MessagePack-encoded); the SPA hydrates from stored history on load so charts are populated immediately. Full details in docs/architecture.md.

04 Security & transparency

Quiet by default. Honest about the edges.

Radical transparency is the whole point: here is exactly what is protected, what gets recorded, and — just as importantly — where the limits are.

Authentication

  • Argon2 password hashing with tuned work factors.
  • Server-side sessions over an HTTP-only cookie.
  • Mandatory email verification; reset links emailed by the API.
  • Optional Google OAuth (PKCE) sign-in.
  • Opt-in TOTP (RFC 6238) with hashed recovery codes.
  • Failed-login tracking + rate limiting (5 tries / 15 min per email).

Data at rest & in transit

  • Column secrets (TOTP seeds, WireGuard device & server private keys) sealed with a key-encryption key (AES-256-GCM).
  • Tunnel traffic encrypted end-to-end by WireGuard's Noise protocol.
  • The KEK is operator-provided and never leaves your box.
  • No content inspection or DPI anywhere in the path.

What it logs ✓ metadata

  • Device lifecycle events (add, online/offline, key rotate, revoke)
  • WireGuard handshake times and endpoint history per device
  • Per-device byte counters and rolled-up bandwidth
  • Sign-ins, the admin audit trail, and access logs
  • Failed logins with source IP and user-agent
  • Retention is operator-tunable — indefinite by default

What it never logs ✕ content

  • No deep packet inspection or traffic-content capture
  • No DNS-query logging
  • No destination IPs or browsing history
  • No third-party analytics, trackers, or telemetry off-box
04.1 Honest caveats
  • Not zero-knowledge. The server generates and stores each device's WireGuard keypair. A client-only key mode was considered and deliberately removed; keys are always stored server-side.
  • Single key-encryption key. The KEK is provided once via env/secret — there's no HSM and no automatic rotation. Lose it and the sealed secrets can't be recovered.
  • Split-tunnel only. Devices route the VPN subnet, not all traffic. ZeroVPN is a private-network VPN, not a full-tunnel or anonymity tool.
  • Rate limiting is per-email. Login throttling keys on the email address, not the source IP.
  • Google sign-in bypasses TOTP. If you require TOTP everywhere, don't enable the Google link.
  • The API runs elevated. Because it brings up the WireGuard interface in its own container, the API holds NET_ADMIN and /dev/net/tun.
05 Get started

Boring. Repeatable. Fast.

Bring your own Linux box with Docker. From clone to a live topology in about fifteen minutes.

# clone and enter
git clone https://github.com/bhadri01/ZeroVPN
cd ZeroVPN

# generate secrets (session, db, KEK)
make setup

# dev stack: real userspace tunnel + hot reload
make up-dev

# open the console

06 FAQ

Honest answers to the obvious questions.

What does ZeroVPN log about my traffic?
Operational metadata only — device lifecycle events, WireGuard handshake and endpoint history, per-device byte counters, sign-ins, and the admin audit trail. It never inspects packet contents, never logs DNS queries, and never records destination IPs or browsing history. Retention is operator-tunable and defaults to indefinite.
Do I need the WireGuard kernel module?
No. The API brings up the tunnel with userspace boringtun baked into its image, in both dev and production — no modprobe, no /lib/modules, no host kernel dependency.
Can I run multiple regions or hubs?
You can register multiple WireGuard hub servers today and manage them from the admin console. Automatic multi-region failover is on the roadmap, not shipped — the site won't pretend otherwise.
How do I back it up?
Bring your own pg_dump of PostgreSQL plus a copy of secrets/ and .env on a schedule. Guard the KEK above all else: without it, the sealed TOTP seeds and WireGuard private keys can't be decrypted.
What about SSO / bring-your-own auth?
Email + password with optional TOTP, plus Google OAuth, ship today. OIDC / SAML for enterprise SSO is on the roadmap.
What's the license?
AGPL-3.0-or-later. Because of the AGPL's §13 network-use clause, if you run a modified version as a network service you must offer its corresponding source to your users. Self-hosting an unmodified copy for yourself or your team has no such obligation.
